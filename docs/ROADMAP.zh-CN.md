# UNU Engine 路线图（按顺序迭代）

更新时间：`2026-04-23`

## 阶段 1：编辑体验与基础稳定性

目标：让日常编辑稳定、可回退、可持续。

- [x] 场景历史快照 + 撤销/重做
- [x] 全局快捷键（保存、撤销/重做、复制、删除、工具切换、播放）
- [x] 状态消息弹窗（可拖动、可关闭）
- [x] 启动器界面（历史项目、示例项目、项目管理）
- [x] 新建项目弹窗（名称/目录参数化）
- [x] 切换工程后 Scene View 自动重载
- [x] 播放态与编辑态工具行为隔离

## 阶段 2：运行时核心能力

目标：具备真实 2D 游戏最小可玩闭环。

- [x] Input 系统（键鼠 + 映射）
- [x] Camera 组件（跟随、缩放、边界）
- [x] Audio 组件（分组、音量）
- [x] UI 组件（Text/Button）
- [x] 脚本化交互（右键交互 + 距离判断 + 可扩展动作）
- [x] 场景切换（门交互示例）

## 阶段 3：内容生产能力

目标：让关卡和资源可以高效持续产出。

- [x] Tilemap（图块层 + 碰撞层）
- [x] Tilemap 图形化编辑子窗口（可视化编辑）
- [x] Prefab 保存与实例化
- [x] 动画编辑与状态机（Idle/Run/Attack）
- [x] 脚本编辑器基础能力（文本编辑/滚动/高亮）
- [ ] Timeline 关键帧高级轨道（更完整的可视化轨道编辑）

## 阶段 4：玩法扩展与脚本系统升级

目标：提升复杂玩法开发效率和可维护性。

- [ ] 脚本模块化加载与热更新机制
- [ ] 交互动作库扩展（对话、任务、库存、触发器链）
- [ ] 更完整的脚本调试工具（断点、日志过滤、上下文查看）
- [ ] 场景事件系统（全局事件总线 + 可视化绑定）

## 阶段 5：性能与工程化

目标：支持中型项目长期迭代。

- [ ] 渲染缓存层完善（分层缓存策略）
- [ ] 碰撞与脚本执行性能面板
- [ ] 资源依赖分析与未引用检测
- [ ] 构建拆包与启动速度优化
- [ ] 自动化回归测试（关键编辑/播放流程）

## 阶段 6：生态与模板

目标：降低新项目启动与团队协作成本。

- [ ] 多示例项目库（平台跳跃/RPG/解谜）
- [ ] 项目模板向导（按类型初始化）
- [ ] 文档体系持续更新（教程 + API + 最佳实践）
- [ ] 版本迁移工具（工程格式升级）
