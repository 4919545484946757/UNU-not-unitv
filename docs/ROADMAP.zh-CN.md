# UNU Engine Starter 路线图（按顺序迭代）

## 阶段 1：编辑体验与基础稳定性（当前）

目标：先让日常编辑稳定、可回退、效率高。

- [x] 场景历史快照 + 撤销/重做
- [x] 全局快捷键（保存、撤销/重做、复制、删除、工具切换、播放）
- [x] 统一中文状态提示（清理历史乱码）
- [x] 关键操作确认弹窗（删除实体、关闭未保存场景）
- [x] 自动保存（可配置间隔）

## 阶段 2：运行时核心能力

目标：具备真实 2D 游戏最小运行能力。

- [x] Camera 组件（跟随、缩放、边界限制）
- [x] Input 系统（键鼠 + 映射表）
- Audio 系统（BGM/SFX、音量分组）
- UI 组件（文本、按钮、锚点布局）

## 阶段 3：内容生产能力

目标：让关卡与内容可持续产出。

- Tilemap（图块层、碰撞层）
- Prefab 变体与嵌套
- 动画状态机（Idle/Run/Attack）
- Timeline 关键帧属性轨

## 阶段 4：物理与脚本升级

目标：让玩法逻辑可扩展、可调试。

- 接入 2D 物理（刚体、触发器、射线）
- 脚本模块加载（替代仅内置注册）
- 脚本错误隔离与调试面板
- 运行时性能采样（FPS/耗时分段）

## 阶段 5：资源管线与发布

目标：支持中型项目协作与交付。

- 图集自动打包与资源压缩
- 资源依赖分析与未引用检测
- 构建 profile（dev/release）
- 一键导出 Web/Electron

## 阶段 6：质量与团队协作

目标：让项目可维护、可长期迭代。

- 序列化回归测试
- 编辑器交互测试（关键流程）
- 项目模板与文档示例
- 版本迁移器（文件格式升级）
